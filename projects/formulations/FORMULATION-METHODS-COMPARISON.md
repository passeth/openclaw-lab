# 처방 현실성 강화 — 방법론 비교 & 최적 설계

> **질문:** 사전 연산 + RPC 룩업이 최선인가? 더 나은 방법은?  
> **결론:** 단일 방법으로는 부족. **3개 레이어를 조합**해야 최적.

---

## 1. 후보 방법론 6가지

### 방법 A: 사전 연산 + RPC 룩업 (앞서 제안)

```
[1회 연산] 공출현/클러스터/프로파일 → Supabase 테이블 저장
[매번] RPC 조회 → AI가 해석 → 처방 생성
```

| 장점 | 한계 |
|------|------|
| 빠름, 구현 간단 | 패턴은 보여주지만 "왜"는 모름 |
| 데이터 변경 시 재연산만 | 복잡한 상호작용 놓침 |
| | AI가 매번 다르게 해석할 수 있음 (일관성 부족) |

**적합도:** ★★★☆☆ — 기반은 되지만 이것만으로는 "교과서 처방"을 완전히 벗어나기 어려움

---

### 방법 B: ML 예측 모델 (Supervised Learning)

```
[학습] 1,254개 BOM + 물성(pH/점도/비중/성상) → 모델 학습
[추론] 원료 조합 입력 → pH/점도/투명도/안정성 예측
```

**가능한 모델:**
- **Random Forest / XGBoost**: 배합비 → pH, 점도 예측 (해석 가능)
- **ANN (신경망)**: 배합비 → 성상 분류 (투명/불투명/크림 등)
- **GNN (그래프 신경망)**: 원료 간 상호작용을 그래프로 모델링 (Intellico 방식)

**EVAS 데이터로 실제 가능한 것:**

| 예측 대상 | 학습 데이터 | 실현 가능성 |
|----------|-----------|:---:|
| pH | 1,254개 중 66% 보유 (~827건) | ✅ 충분 |
| 점도 | 64% 보유 (~802건) | ✅ 충분 |
| 비중 | 86% 보유 (~1,078건) | ✅ 충분 |
| 성상 (투명/불투명) | 76% 보유 (~952건) | ✅ 가능 |
| 안정성 | 데이터 없음 | ❌ 불가 |

| 장점 | 한계 |
|------|------|
| "이 조합의 pH는 5.3일 것" 정량 예측 | 학습 데이터 1,000건 → 정확도 한계 |
| "이 조합은 투명 불가" 사전 판별 | 안정성 데이터 없으면 핵심 예측 못함 |
| 일관된 출력 | 모델 구축/유지 비용 |

**적합도:** ★★★★☆ — pH/점도/성상 예측은 즉시 가능. 안정성은 데이터 축적 필요.

---

### 방법 C: 에이전틱 멀티스텝 검증 (Agentic Verification Flow)

```
[Agent 1: Retriever] EVAS BOM에서 유사 레시피 검색 + 시장 벤치마크
     ↓
[Agent 2: Formulator] 검색 결과 기반 처방 초안 생성
     ↓
[Agent 3: Critic] 초안을 4가지 축으로 검증
     │  ✓ 원료 재고/소싱 가능성
     │  ✓ 성분 비호환성 (cosing_function_contexts)
     │  ✓ 안전 농도 범위 (safety_v2)
     │  ✓ EVAS 사용 이력 vs 신규 원료 비율
     ↓
[Agent 4: Optimizer] 검증 실패 항목 수정 → 재검증
     ↓
[최종 처방] 검증 통과 + 신뢰도 스코어
```

**핵심:** 처방을 만드는 AI와 **검증하는 AI를 분리**. 사람 연구원의 워크플로우와 동일.
- 연구원이 처방 → 팀장이 리뷰 → 수정 → 승인

| 장점 | 한계 |
|------|------|
| **자기 검증** — "내 처방이 맞나" 스스로 확인 | 에이전트 4개 = 비용 ↑ |
| 검증 기준이 명확 (4가지 축) | 실행 시간 ↑ |
| 실패 원인이 추적 가능 | |
| 반복 수정 → 품질 수렴 | |

**적합도:** ★★★★★ — 처방 품질에 가장 직접적 영향. **이것이 핵심.**

---

### 방법 D: Embedding 기반 유사 처방 검색 (RAG)

```
[사전] 1,254개 레시피를 벡터 임베딩
[매번] "투명 SF 어성초 샴푸" → 임베딩 → 유사 레시피 Top-K 검색
      → 가장 유사한 실제 EVAS 처방을 컨텍스트로 제공
```

**evas_product_compositions에 이미 `embedding` 컬럼이 있음!**

| 장점 | 한계 |
|------|------|
| "의미적 유사성"으로 검색 (키워드 매칭보다 정확) | 임베딩 품질에 의존 |
| 자연어로 질의 가능 | 배합비 수치적 유사성은 약할 수 있음 |
| 이미 인프라 존재 (embedding 컬럼) | |

**적합도:** ★★★★☆ — RPC 1(유사 레시피 검색)의 상위 호환. 이미 구조 있음.

---

### 방법 E: 시장 대비 차이 분석 (Gap/Diff Analysis)

```
[사전] 시장 56만 행에서 카테고리별 "표준 배합 프로파일" 추출
[매번] EVAS 처방 초안 vs 시장 표준 → 차이 분석
       → "이 원료는 시장 평균보다 2배 높음 — 의도적인가?"
       → "시장에서 95%가 쓰는 이 원료가 빠졌음 — 의도적인가?"
```

| 장점 | 한계 |
|------|------|
| 시장 맥락에서의 포지셔닝 | 추론 배합비 정확도(low 62%) |
| "다르다"가 의도인지 실수인지 구분 가능 | 시장 평균 ≠ 최적 |
| 혁신과 실수의 경계를 구분 | |

**적합도:** ★★★☆☆ — 보조적. 추론 정확도 개선 후 가치 상승.

---

### 방법 F: 처방 시뮬레이션 (Physics-Informed)

```
[모델] HLB 계산, 유화 안정성 예측, CMC 계산
[매번] 처방의 물리화학적 타당성 사전 검증
       → "이 유화제 조합의 HLB는 12.3 — O/W 안정 범위(8-18) 내"
       → "이 계면활성제 농도에서 CMC 이상 — 세정력 확보"
```

| 장점 | 한계 |
|------|------|
| 물리법칙 기반 → 데이터 없어도 작동 | 모든 원료의 물성치 필요 |
| 안정성 사전 예측 | 복잡한 다성분계에서 정확도 ↓ |
| | 구현 복잡도 높음 |

**적합도:** ★★☆☆☆ — 장기 과제. 단기에는 과투자.

## 2. 최적 조합 — 3-Layer Architecture

단일 방법으로는 안 됨. **3개 레이어를 조합:**

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Layer 1: KNOWLEDGE BASE (기반 지식)                             │
│  ═══════════════════════════════════                             │
│  방법 A (사전 연산) + 방법 D (Embedding RAG)                      │
│                                                                 │
│  • 원료 프로파일 테이블 (EVAS 사용 이력, 안전성, 시장 평균)          │
│  • 공출현 매트릭스 (함께 쓰는/안 쓰는 원료 쌍)                      │
│  • 베이스 포뮬러 클러스터 (제품 유형별 평균 처방)                    │
│  • 임베딩 기반 유사 레시피 검색                                    │
│                                                                 │
│  → 이 레이어가 "EVAS의 경험"을 제공                               │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Layer 2: PREDICTION (예측 모델)                                 │
│  ═══════════════════════════════                                 │
│  방법 B (ML 모델) + 방법 E (시장 차이 분석)                        │
│                                                                 │
│  • pH 예측 모델 (XGBoost, ~827건 학습)                           │
│  • 점도 예측 모델                                                │
│  • 성상 분류기 (투명/불투명/크림/젤)                               │
│  • 시장 표준 대비 차이 분석                                       │
│                                                                 │
│  → 이 레이어가 "만들기 전에 결과를 예측"                           │
│                                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Layer 3: AGENTIC VERIFICATION (검증 플로우)                      │
│  ═══════════════════════════════════════════                     │
│  방법 C (멀티스텝 검증)                                           │
│                                                                 │
│           ┌──────────┐                                          │
│           │ Retriever│ ← Layer 1 (유사 레시피 + 원료 프로파일)     │
│           └────┬─────┘                                          │
│                ▼                                                │
│           ┌──────────┐                                          │
│           │Formulator│ → 처방 초안 생성                           │
│           └────┬─────┘                                          │
│                ▼                                                │
│           ┌──────────┐                                          │
│           │ Predictor│ ← Layer 2 (pH/점도/성상 예측)              │
│           └────┬─────┘                                          │
│                ▼                                                │
│           ┌──────────┐                                          │
│           │  Critic  │ ← 4축 검증                                │
│           │          │   ✓ 원료 EVAS 검증도                      │
│           │          │   ✓ 비호환성                               │
│           │          │   ✓ 안전 농도                              │
│           │          │   ✓ 예측 물성 vs 목표 물성                  │
│           └────┬─────┘                                          │
│                ▼                                                │
│        ┌───────────────┐                                        │
│        │   Optimizer   │ → 실패 항목 수정 → Critic 재검증          │
│        └───────┬───────┘                                        │
│                ▼                                                │
│        ╔═══════════════╗                                        │
│        ║  최종 처방     ║                                        │
│        ║  + 신뢰도 점수 ║                                        │
│        ║  + 검증 리포트 ║                                        │
│        ╚═══════════════╝                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 각 레이어의 역할과 비유

| Layer | 역할 | 비유 |
|-------|------|------|
| **L1: Knowledge** | "이 원료 EVAS에서 써봤나? 시장에선?" | **기억** — 시니어 연구원의 경험 |
| **L2: Prediction** | "이 배합이면 pH 5.3, 점도 4000일 것" | **직감** — "이건 투명 안 될 것 같은데" |
| **L3: Verification** | "원료 재고? 비호환? 안전 농도? 물성 OK?" | **검토** — 팀장의 리뷰 |

**핵심:** 지금까지는 L1도 L2도 L3도 없이, 일반 지식만으로 처방했음.
- L1만 추가해도 "우리 원료 기반"으로 전환
- L2까지 추가하면 "결과 예측" 가능
- L3까지 추가하면 "자기 검증" → 연구원에게 바로 쓸 수 있는 초안

## 4. 구현 우선순위

### Phase 1 (이번 주) — L1: Knowledge Base ★★★★★

**투자 대비 효과가 가장 큼. 즉시 품질 개선.**

| 작업 | 시간 | 효과 |
|------|------|------|
| 원료 프로파일 테이블 생성 | 2시간 | EVAS 766종 원료의 사용 이력 즉시 조회 |
| 공출현 매트릭스 생성 | 3시간 | "같이 쓰는/안 쓰는" 패턴 자동 추출 |
| 베이스 포뮬러 클러스터링 | 2시간 | "SF 샴푸 EVAS 평균 처방" 자동 생성 |
| 임베딩 유사 검색 RPC | 1시간 | 이미 embedding 컬럼 존재, 연결만 |

→ **이것만으로도 다음 처방부터 "AOSP003 모른 채 SLMI 추천" 같은 실수 제거**

### Phase 2 (다음 주) — L2: Prediction ★★★★☆

| 작업 | 시간 | 효과 |
|------|------|------|
| pH 예측 모델 (XGBoost) | 4시간 | "이 배합의 pH는 ~5.3" 자동 예측 |
| 점도 예측 모델 | 4시간 | "이 배합의 점도는 ~5000 cps" 예측 |
| 성상 분류기 | 3시간 | "이 배합은 투명 가능/불가능" 판별 |
| 시장 차이 리포트 | 2시간 | "시장 평균 대비 이 처방의 위치" |

→ **"만들기 전에 결과를 안다" — 시제품 실패율 ↓**

### Phase 3 (2주 내) — L3: Agentic Verification ★★★★★

| 작업 | 시간 | 효과 |
|------|------|------|
| Retriever 에이전트 | 3시간 | L1 자동 조회 |
| Formulator 에이전트 | 2시간 | 검색 결과 기반 초안 |
| Critic 에이전트 (4축 검증) | 4시간 | 자동 검증 + 경고 |
| Optimizer (반복 수정) | 3시간 | 검증 실패 → 자동 수정 → 재검증 |

→ **"사람이 리뷰하기 전에 이미 4번 검증된 처방" — 연구원 신뢰도 ↑**

## 5. 최종 처방 레포트 — 3-Layer 적용 후

```
═══════════════════════════════════════════════════════════════
📋 지성두피 샴푸 PURIFY — 3-Layer Verified Formulation
═══════════════════════════════════════════════════════════════

🔗 L1 참조:
   베이스 레시피: AOSP003 (유사도 0.94) — 어성초 캡슐 스크럽 샴푸
   pH 5.8 | 점도 5,500 | 성상: 녹색 점액상 | 양산 ✅

📊 L2 예측:
   예측 pH:    5.6 ± 0.4  (목표 5.0~5.5 → ⚠️ 약간 높음, 구연산 증량 권장)
   예측 점도:  4,800 cps  (목표 3,000~5,000 → ✅ 범위 내)
   투명 가능:  78% 확률   (⚠️ 어성초 0.5% 시 착색 가능 → 시제 확인)

✅ L3 검증 결과 (4/4 PASS):
   ✓ 원료 검증:  12/15 EVAS 검증 (3종 신규 → 명시)
   ✓ 비호환성:   0건 경고
   ✓ 안전 농도:  전 원료 범위 내
   ✓ 물성 예측:  목표 범위 내 (pH 주의)

┌──────────────────────────────────────────────────────────────┐
│ # │ 원료                │  %   │ EVAS │ 시장평균 │ 검증  │ 비고│
├───┼─────────────────────┼──────┼──────┼─────────┼──────┼─────┤
│ 1 │ Water               │ 76.0 │ 59회 │  67%    │ ✅   │     │
│ 2 │ Na C14-16 Olefin Sul│  7.0 │  확인│   -     │ ✅   │ ①  │
│ 3 │ Acrylates Copolymer │  5.0 │  확인│   -     │ ✅   │     │
│ 4 │ Betaine             │  3.0 │142회 │  3.2%   │ ✅   │     │
│ 5 │ K. Cocoyl Glycinate │  2.0 │  확인│   -     │ ⚠️   │ ②  │
│ 6 │ 어성초 추출물         │  0.5 │  확인│  0.1%   │ ⚠️   │ ③  │
│ 7 │ Menthol             │  0.2 │  확인│  0.2%   │ ✅   │     │
│ ...│                    │      │      │         │      │     │
│15 │ 페퍼민트 추출물       │ 0.01 │  확인│   -     │ ✅   │ 마케팅│
└──────────────────────────────────────────────────────────────┘

📝 주의사항 (자동 생성):
① Na C14-16 Olefin Sulfonate: Sulfonate(설포네이트) ≠ Sulfate(설페이트)
   → "Sulfate-Free" 클레임 가능 여부 바이어와 확인 필요
   → 엄격한 SF 기준에서는 불가로 판정되는 경우 있음

② K. Cocoyl Glycinate: AOSP003에서 0.72% 사용
   → 2.0%는 EVAS 최고 사용 농도의 2.8배 → 소량 시제 권장
   → 공출현 데이터: Betaine과 함께 사용 이력 있음 ✅

③ 어성초 0.5%: EVAS 최대 사용 농도 0.01%의 50배
   → L2 예측: 투명도 78% → 착색 리스크
   → 시장 평균: 0.1% → 0.5%도 "고농축" 맞음
```
